# BRAM-Oriented DW Tile Buffer

## Purpose

`rtl/cnn/dw_tile_buffer_bram.v` is the BRAM-backed replacement candidate for
the existing LUT-heavy `dw_tile_buffer.v`. The old module remains connected to
the current DSBlock path until the streaming DSBlock refactor is verified.

The buffer stores:

```text
64 pixels x 128 channels x 8 bits = 65536 bits
```

## Memory Layout

Eight consecutive pixels of one channel form one 64-bit word:

```text
word_addr = {pixel_idx[5:3], channel_idx[6:0]}
byte_lane = pixel_idx[2:0]
```

This produces a `1024 x 64-bit` memory:

```text
word[pixel_group][channel][byte_lane]
    = feature[pixel_group * 8 + byte_lane][channel]
```

The DW path writes one int8 element with byte enable. The PW path reads eight
pixels of one channel in one cycle.

## Interface Contract

- A write occurs on the rising edge when `wr_en=1`.
- `rd_pixel_base` must be a multiple of eight.
- A read request is accepted on the rising edge when `rd_en=1`.
- `rd_valid` and `rd_data_vector` are available after that edge, giving a
  fixed one-cycle synchronous read.
- Read and write ports may operate in the same cycle on different words.
- The DSBlock controller must prevent same-word read/write collisions.
- Reset clears only `rd_valid` and the read output register. Stored feature
  data is not cleared.

## Implementation

Vivado synthesis uses `xpm_memory_sdpram` with:

```text
MEMORY_PRIMITIVE   = block
MEMORY_SIZE        = 65536
PORT A             = 1024 x 64, byte write enable
PORT B             = 1024 x 64, synchronous read
READ_LATENCY_B     = 1
CLOCKING_MODE      = common_clock
```

Verilator uses a behaviorally equivalent `1024 x 64-bit` array because XPM is
not available in the simulator.

## Verification

`tb_dw_tile_buffer_bram` uses cycle-level vectors generated by
`generate_dw_tile_buffer_bram_cases`. It covers:

- all 1024 word addresses;
- all eight byte-write lanes;
- 8192 scalar DW writes;
- back-to-back PW reads;
- simultaneous reads and writes on different words;
- reset preservation;
- pixel group 7 and channel 127.

Current result:

```text
PASS tb_dw_tile_buffer_bram cycles=9355
```

## Vivado 2021.2 OOC Result

Target: `xck26-sfvc784-2LV-c`

| Resource | Old buffer | BRAM buffer |
| --- | ---: | ---: |
| CLB LUT | 46131 | 10 |
| CLB register | 64 | 1 |
| LUT as memory | 1280 | 0 |
| RAMB36E2 | 0 | 2 |
| DSP | 0 | 0 |

The installed K26 part remains unable to provide timing analysis, so this
result proves synthesis and memory mapping, not timing closure. Vivado also
reports that a second output register could improve BRAM read timing; the
current design intentionally keeps the required one-cycle read latency.
